QLscript 是一个面向青龙面板的个人自动化脚本库。任务脚本集中放在 `scripts/`，公共方法放在 `utils/`，模板和维护标准放在 `templates/` 与 `SCRIPT_STANDARD.md`。

## 仓库约定

- `scripts/` 只放青龙实际执行的任务脚本。
- `utils/` 存放 Python / JavaScript 公共工具。
- `templates/` 是新脚本模板，不作为任务执行。
- `README.md`、`scripts/README.md`、脚本头和代码读取的环境变量名必须保持一致。
- 日志和通知不得输出完整 token、cookie、Authorization 或完整请求头。

## 使用顺序

第一次接入时先看快速开始和环境变量。已经能运行后，再按脚本详解页补充每个任务的 Cookie、token 或请求体。新增脚本时从开发规范开始，并同步脚本索引、脚本详解页和根 README。
