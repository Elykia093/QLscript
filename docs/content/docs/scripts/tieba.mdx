## 基本信息

| 项目 | 内容 |
| --- | --- |
| 文件 | `scripts/tieba.py` |
| cron | `30 8 * * *` |
| 任务名 | `百度贴吧` |
| 依赖 | Python `requests`、青龙 `notify.py` |
| 环境变量 | `TIEBA_COOKIE` |

## 环境变量

`TIEBA_COOKIE` 可以填完整 Cookie，也可以只填 BDUSS。多账号优先换行分隔。

```text
BDUSS=xxx
BDUSS=yyy
```

## 执行流程

1. 从 Cookie 中提取 BDUSS。
2. 获取 `tbs`。
3. 拉取关注贴吧列表，并按贴吧 id 去重。
4. 并发签到关注贴吧，失败贴吧最多重试 5 轮。
5. 通知中展示成功/已签数量和最多 8 个失败贴吧。

## 常见失败

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 未找到 BDUSS | 变量为空或 Cookie 缺少 BDUSS | 重新填写完整 Cookie 或 BDUSS |
| 未获取到关注贴吧 | 账号没有关注贴吧，或 Cookie 失效 | 登录贴吧检查关注列表 |
| Cookie 失效或需要验证 | 百度触发风控或登录态过期 | 手动登录验证后重新获取 Cookie |

## 运行特点

脚本会限制并发数量，并在单个贴吧签到前加入随机延迟。这样可以降低连续请求造成的风控风险，但不能保证绕过平台验证。
