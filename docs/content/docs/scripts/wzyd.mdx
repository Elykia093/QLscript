## 基本信息

| 项目 | 内容 |
| --- | --- |
| 文件 | `scripts/wzyd.js` |
| cron | `40 8 * * *` |
| 任务名 | `王者营地` |
| 依赖 | Node.js 18+、青龙 `sendNotify.js` |
| 环境变量 | `WZYD_HEADERS`、`WZYD_BODY` |

## 环境变量

`WZYD_HEADERS` 填签到请求头 JSON 对象，`WZYD_BODY` 填签到请求体 JSON 对象。两个变量的账号数量必须一致；数组、字符串和 `null` 会在请求前被拒绝。

旧变量 `WZYD_TOKEN` 仍兼容读取，但新配置应统一使用 `WZYD_HEADERS`。同时配置两者时，以 `WZYD_HEADERS` 为准。

```text
{"cookie":"xxx","user-agent":"xxx"}
{"cookie":"yyy","user-agent":"yyy"}
```

```text
{"roleId":"123","areaId":"1"}
{"roleId":"456","areaId":"1"}
```

## 执行流程

1. 按账号拆分请求头和请求体。
2. 校验每个账号是合法 JSON 对象。
3. 调用王者营地签到接口。
4. 使用 `roleId` 脱敏值作为通知标题。

## 常见失败

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 请求头不是合法 JSON | `WZYD_HEADERS` 格式不完整 | 用 JSON 校验工具检查每一行 |
| 请求体不是合法 JSON | `WZYD_BODY` 格式不完整 | 确认引号、逗号和花括号完整 |
| 请求头或请求体不是对象 | 配置成了数组、字符串或 `null` | 改为 `{...}` 形式的 JSON 对象 |
| 账号数量不一致 | 两个变量拆分后的行数不同 | 保持请求头和请求体一一对应 |
