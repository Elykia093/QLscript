## 基本信息

| 项目 | 内容 |
| --- | --- |
| 文件 | `scripts/mihoyo.py` |
| cron | `0 9 * * *` |
| 任务名 | `米游社` |
| 依赖 | Python `requests`、青龙 `notify.py` |
| 环境变量 | `MIHOYO_COOKIE`、`MIHOYO_GIDS` |

## 环境变量

`MIHOYO_COOKIE` 填米游社 Cookie。`MIHOYO_GIDS` 可选，用于指定社区分区，默认 `2,6,8`。

| gid | 社区 |
| --- | --- |
| `1` | 崩坏3 |
| `2` | 原神 |
| `3` | 崩坏2 |
| `4` | 未定事件簿 |
| `5` | 大别野 |
| `6` | 崩坏：星穹铁道 |
| `8` | 绝区零 |
| `9` | 崩坏：因缘精灵 |

多个 gid 可用逗号、中文逗号或空白分隔；重复 gid 只会签到一次。

```text
2,6,8
```

## 执行流程

1. 读取目标 gid 列表。
2. 为每次请求生成 DS、设备 ID 和移动端请求头。
3. 逐个分区签到，分区之间加入 1 到 2 秒随机延迟。
4. 通知中按分区汇总返回消息。

## 常见失败

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| `MIHOYO_GIDS` 未配置有效分区 | gid 填错或不在支持列表中 | 按上表修正 |
| 触发验证码 | 平台风控 | 手动处理验证，降低运行频率 |
| Cookie 失效 | 登录态过期 | 重新获取 Cookie |
