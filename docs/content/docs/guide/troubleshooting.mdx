---
title: 排障
description: 常见运行失败、依赖缺失和账号异常的排查顺序。
---

## 依赖缺失

如果 Python 脚本提示 `ModuleNotFoundError`，先确认青龙环境已安装 `requests`。如果 JavaScript 脚本启动失败，确认 Node.js 版本不低于 18。

## 环境变量为空

脚本应在启动阶段校验必填环境变量。遇到变量为空时，先检查青龙变量名是否和文档一致，再检查变量是否启用。

## Cookie 或 token 失效

登录态失效通常表现为接口返回未登录、权限不足或业务错误码。优先重新获取凭据，并确认日志没有输出完整敏感内容。

## 多账号部分失败

单个账号失败不应影响其他账号。排查时先看失败账号的序号和关键原因，再确认该账号的 Cookie、token 或请求体是否独立有效。

## 通知没发送

通知模块缺失时脚本会降级输出到控制台。确认青龙环境中存在 `notify.py` 或 `sendNotify.js`，再检查通知平台配置。
