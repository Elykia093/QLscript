title: Cookie 与凭据
description: Cookie、token 和请求头的填写与脱敏规则。
---

## 填写原则

- 凭据只放在青龙环境变量里，不写死到代码或文档示例里。
- 日志和通知只能输出账号序号、脱敏账号标识和关键错误原因。
- 不打印完整 token、cookie、Authorization 或完整请求头。
- 多账号 Cookie 默认使用换行分隔，避免把 Cookie 自身的分号当成账号分隔符。

## 请求头 JSON

部分脚本需要保存请求头或请求体 JSON，例如 `WZYD_HEADERS` 与 `WZYD_BODY`。这类变量必须是 JSON 对象，账号数量也要一一对应。多账号优先使用换行分隔；公共拆分器会先识别完整 JSON，避免单账号内容里的 `&` 或 `#` 被误切。旧变量 `WZYD_TOKEN` 仍可读取，但新配置统一使用 `WZYD_HEADERS`。

## 排查建议

如果脚本提示登录失效，优先重新抓取 Cookie 或 token。确认变量内容没有被青龙自动换行、截断或转义后，再判断脚本逻辑是否需要更新。
